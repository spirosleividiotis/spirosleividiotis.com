Approach

Production at Scale

Every animation followed a consistent process: design with constraints, optimize for performance, test across platforms, document for engineers. I wasn't just animating—I was building a production pipeline.

Lottie files were structured for reusability. Colors and values were tokenized where possible. File sizes were optimized (most under 50KB). Animations were tested on real devices, not just design tools.

The library covered everything from button states and loaders to onboarding flows and success celebrations. Each animation was purposeful, polished, and ready to ship.

Impact

Beyond Individual Files

This library wasn't just a collection of animations—it was a production asset. Engineers could integrate Lotties knowing they'd perform well. Designers could reference existing work before creating new animations. Product teams could ship features with polished motion without custom requests.

The consistency across the library reinforced Qonto's brand. Every animation felt like part of the same system—purposeful, refined, and ready for production.
