Qonto Motion System

Qonto has a strong design system, and we wanted animation to be part of it. This is the story of how we built a scalable motion language that bridges design and engineering—from principles to implementation.

2022–2024 · Qonto · Motion Design Lead
Context

The Problem

Qonto's design system provided clarity for color, typography, spacing, and components. But motion? It was inconsistent. Designers mocked up animations with arbitrary values. Engineers implemented them differently across platforms. There was no shared vocabulary.

We needed a motion system that would scale—one that designers could reference in specs, engineers could implement without ambiguity, and teams could extend without breaking consistency.

Approach

Building the System

01

Motion Principles

We started by defining motion principles—the foundation for every animation decision. These principles guide what good motion feels like at Qonto: purposeful, coherent, subtle, snappy, and smooth.

Each principle came with visual examples showing do's and don'ts, helping teams understand not just the rules, but the reasoning behind them.

02

Primitives: Speaking the Same Language

Next, we worked with engineers to define primitive tokens—the building blocks of motion. We asked them what they needed: duration values, easing curves, scale factors, blur intensities.

We talked to designers about how they spec'd motion in mockups. The result: a shared vocabulary. motion.duration.quick means 200ms everywhere—Figma, iOS, Android, Web. No interpretation. No ambiguity.

Primitives include: Duration, Easing, Scale, Alpha, Blur, Rotation, Translate, and Delay.

03

Semantics: Scalable Behaviors

Primitives alone weren't enough. Designers don't think in milliseconds and cubic-beziers—they think in behaviors. "This button should feel responsive." "This modal should slide in smoothly."

So we created semantic tokens—predefined combinations of primitives for common patterns. Behaviors like Fade, Slide, Press/Tap, Hover/Lift, and Expand/Collapse.

These semantics are organized into Base (entry, exit, reveal, transition) and Common (interactions, micro-animations). They're documented with specs, code snippets, and live previews.

Impact

The Result

Implementation with the motion system is faster. Designers know what behaviors to use from the library. Engineers reference tokens in code—no guesswork, no back-and-forth.

We scale. New features ship with consistent motion because the patterns are already defined. We're consistent across web and mobile because we share the same source of truth.

Most importantly, we have a common motion language. Designers, engineers, and product teams can discuss animation with precision, using the same vocabulary.
