Motion Specifications

  • For each state:
    • What are the transition animations? (e.g., fade-in loader, skeleton shimmer, error shake)
    • Timing/duration (ms), easing curves, sequencing
    • Accessibility considerations (motion reduction, focus management)
  • Provide motion rationale (why use this motion, what’s the UX value?)

What to Include in Documentation

Must Have:

  • Rationale (the “why”)
  • Visual references (mockups, motion, Figma components)
  • Usage guidelines and specs (timing, motion)
  • Accessibility

Nice to Have:

  • Competitive analysis
  • User feedback or research insights
  • Edge cases and troubleshooting
