üîß Duration

  • Shopify Polaris uses a discrete scale: 50ms‚Äì500ms in consistent 50ms increments
  • Material Design 3 groups tokens into semantic categories like fast, standard, slow; each with defined ms values

Takeaway: Use a balanced scale of durations that supports both micro-interactions and narrative-level motion.

‚ÜîÔ∏è Easing

  • Polaris includes: ease-in, ease-out, ease-in-out, linear, plus an ease custom default designed for response-style interactions
  • Material Design favors expressive cubic-bezier curves with emphasis on smooth stop/start and longer duration for polish

Takeaway: Use custom easing curves tied to system tokens, favoring B√©zier values over CSS keywords.

‚è≥ Delay

  • Shopify and others use delay tokens sparingly‚Äîprimarily to enable sequencing or stagger of multi-element motion

Takeaway: Define delay tokens (e.g., 50ms, 100ms) and reinforce their use in choreographed flows like tooltips and onboarding panels.

‚ÜïÔ∏è Scale & Rotation

  • While Polaris doesn‚Äôt expose scale/rotation tokens by default, other systems embed these into keyframe token utilities
  • Atlassian provides predefined motion modules (e.g. resizing, toggles, entering elements) where scale and rotation play a role

Takeaway: Include scale and rotation tokens even if simpler‚Äîthey provide clarity for icon toggles, feedback interactions, and spinner behavior.

‚úÖ System Design Recommendations

  • Follow Polaris-style naming conventions: motion-duration-100, motion-easing-standard, motion-delay-50
  • Descriptive purpose over arbitrary numbers: use tokens like -standard, -emphasized, -decelerate to clarify intent
  • Ensure consistency across platforms and include auto conversion tools (as done in Polaris migration scripts)
  • Support accessibility by planning for a duration-scalar or reduced motion override, per Norton example

General Learnings from the Page

1.

Structure Matters

The database separates motion into digestible layers, showcasing how teams approach:

  • Principles: Why motion is used, and its philosophical role in the brand.
  • Tokens & Specs: Concrete values (like timing, easing) with naming conventions.
  • Usage: Real examples of how motion is applied in components or transitions.

You can replicate this structure in your A3:

1. Principles
2. Motion Tokens
   - 2.1 Naming
   - 2.2 Values (Duration, Easing, etc.)
   - 2.3 Application ("Where/How")
3. Component Behaviors / Interaction Specs
