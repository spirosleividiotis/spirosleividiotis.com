Naming

Context

Naming motion tokens consistently is key to building a scalable and intuitive design system. Good names make tokens easy to use, easy to maintain, and future-proof as the system grows. There are multiple valid approaches from semantic and size-based to numeric scales. At Portfolio, we explored three naming conventions to determine the best balance between clarity, scalability, and alignment with our existing token strategies (e.g. colors, spacing).

Naming convention

At Portfolio, we evaluated three naming conventions to determine the best balance between:

1. Size-based→ motion-duration-m

This approach uses abstract size labels (like s, m, l, or xs, xl) to communicate the relative duration of a motion. It is human-friendly and quickly conveys the idea of small vs large or short vs long animations. This strategy works well when there are only a few motion values and when semantic use cases are clear.

2. Numeric scale → motion-duration-1

This option assigns a simple incremental number to each motion duration token. It creates a clear order and is scalable as more tokens are added. However, it hides the actual value (e.g. milliseconds) and offers no intrinsic meaning to users.

3. Numeric scale → motion-duration-100

This approach uses numbers that correspond to approximate millisecond values, like 100 for 100ms, 250 for 250ms, etc. It’s highly transparent and aligns perfectly with how Portfolio names color tokens (e.g. gray-100, gray-500). The naming communicates both hierarchy and intent while offering flexibility for semantic aliases if needed later.
