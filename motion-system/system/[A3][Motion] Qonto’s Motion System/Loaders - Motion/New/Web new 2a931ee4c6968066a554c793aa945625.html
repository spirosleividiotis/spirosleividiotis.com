Skeleton loader

Best practices

A Skeleton loader should give real-time information of what is being loaded in the different blocks of the app that have a constant layout. It’s the closest to showing real content and should be preferred whenever possible

Anatomy

Structure

Use these elements to create the layout of your skeleton. You can adapt the size of the elements to fit your use case.

Skeleton elements

  1. Text
  1. Avatar
  1. Tag
  1. Button

Indeterminate spinner

Best practices

Indeterminate spinners help visualizing variable short loading times (in context) that will be understood by users as the direct result of their action.

Anatomy

Structure

Spinners come in various sizes for different situations.

Spinner anatomy

  1. Large spinner : fullscreen loader
  1. Medium (mobile only) : button loader
  1. Small : Large button loader
  1. Extra-small : Medium / small button loader

Lottie animation loader

Best practices

If you know in advance that a sensitive action will result in a longer than average loading time (around 10s+), dedicate a proper page with enough context to help the user understand what’s going on.

Determinate progress bar

Best practices

As opposed to an indeterminate spinner, a determinate progress bar or a determinate spinner give exact feedback of the progression. The size and context is well defined therefore users will understand what the percentage means without explicitly saying that “the file is currently uploading”.

Anatomy

Structure

Progress bar

  1. Progression bar
  1. Background bar

Behavior

Defined length

A progress bar loader should have a clearly defined beginning and end to give context while it fills progressively. In the case of loading states, the color should remain a primary stroke color as it only informs about back end processing.
