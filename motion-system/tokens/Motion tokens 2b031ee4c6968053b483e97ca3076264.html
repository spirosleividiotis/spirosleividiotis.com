What are Motion tokens?

Motion tokens are the foundation of a scalable Motion System at Portfolio.
They are divided into two main groups:

Tokens help bridge design and development by using shared language and structure, enabling smoother handoff and easier maintenance.

Primitive tokens

Timing

Core time controls for any animation.

Values

Visual change primitives applied to elements.

Semantic tokens

Base

Purpose-driven combinations of timing primitives with no visual change.
Designers pick by intent - engineers apply effects as needed or use Common patterns.

Common

Ready-made patterns combining Base timing with effect(s).

Motion Decision Guide

Quick start: Pick Common tokens first - they cover most use cases. If you need something custom, combine a Base token with Values.

How to choose:

  • Start with Common tokens → Use the sections below to find your match.
  • Review the token details → Each token has Purpose, Use/Don't, Properties, Preview, and Code alias.
  • Customize if needed → Pair a Base token (entry, exit, reveal, transition)
    with Value primitives.

Customize if needed

If no Common token fits your needs:

  1. Pick a Base token (motion-entry, motion-exit, motion-reveal, motion-transition)
  1. Combine it with Value primitives (motion-translate, motion-scale, motion-alpha, etc.)

Token Structure

Motion tokens follow a 3-tier hierarchy:

1. Primitives → motion-duration-200, motion-scale-95, motion-alpha-0

  • Building blocks for timing and visual changes

2. Base → motion-entry-default (duration + easing + delay)

  • Purpose-driven timing combinations (no visual change)

3. Common → motion-slide-up (Base + Values)

  • Ready-made patterns combining timing with effects
