After building Qonto's motion system, designers and engineers needed a way to see and test motion tokens in context. I built an internal tool that lets anyone preview how real product designs—modals, buttons, transitions—would look with actual motion values. No guessing, no back-and-forth. Just apply, preview, ship.

2024 Qonto Staff Motion Designer, Tooling
Context

The Problem

We had motion tokens—durations, easings, scales—documented and implemented across platforms. But there was a gap: no one could see them in action without building a prototype or shipping code.

Designers would mock up animations using arbitrary values. Engineers would ask, "Which easing should this use?" PMs would struggle to visualize motion behavior during reviews. Every conversation required mental translation: "Imagine this fading in over 300ms with ease-out."

We needed a live preview tool—a way to apply motion tokens to real product designs and see exactly how they'd behave.

Approach

Building the Tool

01

Basic Token Inspector

The first version was simple: a library of all our motion tokens with live previews. Select motion.duration.quick, see a box animate for 200ms. Pick motion.easing.standard, watch the easing curve in action.

Designers could finally feel the difference between 200ms and 400ms, between ease-out and spring. Engineers could reference exact token names. Everyone spoke the same language.

02

Previewing in Product Context

Seeing tokens in isolation was useful, but we needed more: how does this modal entrance feel? How does this button press look with our scale token?

I added product templates—common patterns like modals, sheets, toasts, button states. Designers could upload their Figma frames, apply motion tokens, and preview the result. No code, no prototyping tools, just instant feedback.

This became the bridge between static designs and implemented motion. A designer could show a PM, "Here's how this will animate," and they'd both see the same thing an engineer would ship.

03

Advanced Mode: Timeline & Stagger

The basic tool covered 80% of cases. But for complex sequences—staggered lists, multi-step onboarding, orchestrated transitions—we needed more control.

I built an advanced timeline mode: layer multiple animations, adjust delays, preview stagger patterns. Designers could prototype complex choreography using system tokens, then hand engineers a spec that was already technically accurate.

It wasn't After Effects—it was better. Every value was a token. Every animation was implementation-ready.

Impact

The Result

The tool became the source of truth for motion at Qonto. Designers used it in their workflow. Engineers referenced it during implementation. PMs could see motion behavior before features shipped.

More importantly, it scaled motion beyond me. Product designers could apply motion tokens without asking me. Engineers could self-serve token values. New team members could onboard by exploring the tool.

Motion went from a bottleneck to a shared capability. The tool didn't just document our system—it made it accessible, usable, and real.
