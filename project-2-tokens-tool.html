After building Qonto's motion system, designers and engineers needed a way to see and test motion tokens in context. I built an internal tool that lets anyone preview how real product designs—modals, buttons, transitions—would look with actual motion values. No guessing, no back-and-forth. Just apply, preview, ship.

Created an internal tool that lets designers preview motion tokens on real product designs—modals, buttons, transitions. Upload a Figma frame, apply tokens, see the result. Built with Cursor AI, made the motion system accessible to everyone.

Context

The Problem

Motion tokens existed—documented, implemented—but no one could see them in action without building a prototype or shipping code. Designers used arbitrary values. Engineers asked which easing to use. Every conversation required mental translation.

Approach

Building the Tool

01

Basic Token Inspector

A library of all motion tokens with live previews. Select a token, see it animate. Designers could finally feel the difference. Engineers got exact names. Everyone spoke the same language.

Token Inspector Interface

Build motion, get code for iOS, Android, CSS, and visual specs

02

Previewing in Product Context

Added product templates—modals, sheets, toasts. Designers upload Figma frames, apply tokens, see the result. The bridge between static designs and implemented motion.

Product Context Preview

Upload an asset and compare different motion tokens side-by-side

03

Advanced Mode: Timeline & Stagger

For complex sequences—staggered lists, orchestrated transitions—built an advanced timeline mode. Layer animations, adjust delays, preview stagger patterns. Every value a token. Every animation implementation-ready.

Initiative

Empowering Designers to Build Tools

This tool proved a bigger idea: designers could ship their own tools using AI. I created Cursor AI templates and guidelines—HTML templates, design tokens, detailed prompts—so anyone at Qonto could build production-ready internal tools.

The Motion Tokens Inspector became the team's shared resource for motion decisions. Designers preview tokens independently. Engineers get exact values. The system scaled beyond me.

Impact

The Result

Motion went from a bottleneck to a shared capability. Product designers apply tokens independently. Engineers self-serve values. New team members onboard by exploring. The tool didn't just document the system—it made it accessible, usable, and real.
